# ☁️ ОБЛАКО — Адаптивная Социальная Платформа

Современная, легковесная и безопасная социальная платформа для общения, обмена контентом и взаимодействия в реальном времени. Проект полностью контейнеризирован с помощью Docker и оптимизирован для работы на любых устройствах.

## Фичи проекта

*   **Адаптивный UI/UX**: Интерфейс разработан по принципу Mobile-First. Навигация, лента и чаты адаптируются под экраны смартфонов, обеспечивая опыт нативного приложения.
*   **E2E-безопасность переписки**: Встроенная система генерации криптографических ключей при регистрации пользователя для защиты приватных чатов.
*   **Умная лента контента**: Динамическая сортировка постов по популярности, новизне и количеству обсуждений.
*   **Персонализация профилей**: Кастомизация пользовательских баннеров с помощью встроенного инструмента выбора цвета (пипетки).
*   **Real-time инфраструктура**: Чаты и уведомления работают мгновенно на базе протокола WebSockets (Socket.io) с кастомными маршрутами.

---

## Технологический стек

*   **Frontend**: React, TypeScript, Tailwind CSS, Lucide Icons.
*   **Backend**: Node.js, Express, TSX, WebSockets (Socket.io).
*   **Database & ORM**: PostgreSQL, Prisma ORM.
*   **Инфраструктура**: Docker, Docker Compose (полная изоляция сервисов).

## Быстрый запуск проекта в Docker

Проект полностью настроен для развертывания одной командой. Вам не нужно устанавливать PostgreSQL или Node.js локально.

### 1. Клонирование репозитория
```bash
git clone https://github.com
cd oblako
```

### 2. Настройка окружения
Создайте файл `./backend/.env` и укажите конфигурацию базы данных и секреты сессий:
```env
POSTGRES_USER=postgres
POSTGRES_PASSWORD=admin123
POSTGRES_DB=hacaton

DATABASE_URL="postgresql://postgres:admin123@postgres-db:5432/hacaton?schema=public"
SESSION_SECRET="your_generated_crypto_secret_here"
```

### 3. Развертывание и запуск инфраструктуры
Запустите сборку и старт всех контейнеров в фоновом режиме:
```bash
docker compose up -d --build
```

### 4. Синхронизация схемы базы данных (Prisma)
После того как контейнеры успешно запустятся, накатите таблицы в чистую базу данных PostgreSQL:
```bash
docker compose exec backend npx prisma db push
```

Готово! Приложение доступно в браузере по адресу: **`http://localhost`**

## 🗂 Архитектура контейнеризации (Docker Compose)

*   `cloud_frontend`: Раздает адаптивное SPA-приложение React. Доступен снаружи на стандартном 80-м порту.
*   `cloud_backend`: Серверное API на Node.js, обрабатывающее HTTP-запросы и WebSocket-соединения на порту 3000.
*   `cloud_postgres`: Реляционная база данных PostgreSQL (порт 5432).
*   **Docker Volumes**: Использованы именованные тома (`postgres_data`) и синхронизация директории файлов (`uploads`) для полной сохранности данных пользователей, постов и аватарок при перезапуске или удалении контейнеров.
